🚀 Platform Features

Everything VoltDrive
Offers

A complete suite of real-time tools for EV drivers finding their next charge and planners deciding where infrastructure is needed most.

🌐
Powered by Live OpenChargeMap API
Every prediction, station listing, and recommendation on VoltDrive is derived from live data fetched directly from OpenChargeMap — the world's largest open EV charging network database. Smart 6-hour caching keeps the experience fast even if the API is temporarily unreachable.
📍
Nearby Station Finder
Locate the closest available EV charging stations in real time using your GPS coordinates. Results are sorted by walking or driving distance, with live port availability shown for each station. Filter by port type — Fast, Medium, or Slow — and set a search radius of up to 30 km.
GPS-based Live availability Up to 30 km radius
🔮
Future Station Prediction
A multi-factor greedy algorithm evaluates all 14 Kerala districts against 7 scoring criteria derived entirely from live OCM data. Districts are ranked by need score and the system identifies which areas require new DC fast chargers, AC fast chargers, or slow chargers — with full explanations.
14 districts 7 scoring factors Greedy algorithm
⚡
Real-Time Port Availability
Port status is updated every 15–45 seconds using time-aware simulation models. Morning and evening rush hours (8–10 AM and 6–8 PM) trigger higher station utilisation, while off-peak hours show improved availability — reflecting real charging behaviour across Kerala's EV fleet.
Updates every 15–45s Rush-hour aware Live utilisation %
🗺️
OSRM Navigation
Turn-by-turn driving navigation is powered by OSRM (Open Source Routing Machine). When you select a charging station, VoltDrive calculates the accurate road distance and estimated travel time from your current location — no third-party map subscription required.
OSRM routing Drive time estimate Open source
🔌
Multi-Connector Support
VoltDrive supports all major Indian EV connector standards. DC fast charging via CCS2 and CHAdeMO for cars and buses, AC fast charging via Type 2 (IEC 62196) for cars and vans, and AC slow charging via the Bharat AC-001 standard for two-wheelers and three-wheelers.
CCS2 / CHAdeMO Type 2 IEC 62196 Bharat AC-001
📊
Demand Analysis
Each district is classified as High, Medium, or Low demand based on EV density, station count, port availability, operator reliability, and growth indicators all derived from live data. High-demand districts with zero infrastructure are flagged as urgently needing immediate deployment.
High / Medium / Low Live data driven District-level
🛡️
SDN-Based Scheduling
Software Defined Networking principles centralise the intelligence that normally lives inside each individual charger. The SDN controller has a global view of all stations — their capacity, demand, operator, and connectivity — enabling optimised charging session allocation and load balancing across the network.
SDN architecture Load balancing Centralised control
🔄
Smart Caching & Refresh
Live API data is cached locally for 6 hours to ensure fast page loads even during API downtime. A background refresh thread automatically re-fetches fresh data every 6 hours without any disruption. If live data is unavailable, a validated fallback dataset maintains full functionality.
6-hour cache Auto-refresh Fallback dataset
